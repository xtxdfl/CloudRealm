主机管理平台功能设计

一、核心定位

面向服务器集群的中心化主机管控平台，基于 Agent 实现主机全生命周期管理，支持单台 / 批量运维操作，实时同步主机状态，支撑业务服务器统一管控。

二、主机状态定义（核心优化：状态闭环，无歧义）

统一管理主机全生命周期状态，所有操作严格遵循状态流转：

未初始化：录入信息，未部署 Agent

部署中：执行添加主机流程

运行中：正常提供服务

停止中：执行停止操作

已停止：主机 / Agent 停止运行

维护中：业务暂停，禁止常规启停

下线中：执行下线流程

已下线：Agent 卸载，保留记录

异常：流程失败 / 心跳断开 / 操作超时

三、核心功能模块（完整流程 + 优化细节）

1\. 主机基础管理

（1）添加主机（部署 Agent）

核心目标：完成目标主机 Agent 自动化部署，实现平台与主机互通执行流程（标准化）：

参数校验：校验 IP、端口、账号、密钥等必填项，IP 唯一性校验

网络连通性检测：ping + SSH 端口探测

SSH 免密配置：本地生成公钥 → 推送至目标主机 → 免密验证

本地环境监测：OS 版本、CPU / 内存 / 磁盘、端口占用、系统权限

依赖安装：自动安装 Python/JDK/ 系统库等 Agent 运行依赖

Agent 服务部署：上传安装包 → 解压安装 → 注册系统服务（systemd）

服务启动与自启配置：启动 Agent，设置开机自启

互通性测试：Agent 心跳上报、指令下发验证

状态更新：部署成功→运行中；失败→异常，记录失败原因

支持能力：单台添加、批量导入添加（Excel / 模板）

（2）删除主机

核心目标：安全清理主机资源，删除数据库记录执行流程（严格顺序）：

前置校验：仅允许删除已下线/异常状态主机

执行维护：停止相关业务，标记维护状态

执行下线：卸载 Agent、清理安装文件、取消免密

数据清理：删除数据库主机记录、关联配置、历史日志

最终状态：物理删除数据，不可恢复

2\. 主机运维操作（单台 + 批量，统一能力）

所有启停 / 重启操作支持批量选择、任务异步执行、进度实时查看

（1）启动主机

适用状态：已停止、异常、维护中

能力：单台启动、批量启动（支持勾选多台）

流程：下发启动指令 → Agent 启动 → 心跳检测 → 更新为运行中

（2）停止主机

适用状态：运行中、维护中

能力：单台停止、批量停止

流程：下发停止指令 → 优雅停止 Agent → 状态更新为已停止

安全策略：支持强制停止 / 优雅停止

（3）重启主机

适用状态：运行中、已停止

能力：单台重启、批量重启

流程：停止 → 启动 → 心跳验证 → 状态同步

3\. 主机状态管理

（1）主机维护

适用场景：系统升级、硬件检修、业务调试

能力：单台 / 批量进入维护模式

效果：

状态更新为维护中

禁止常规启停、重启操作

保留 Agent 连通性，支持运维指令

支持退出维护，恢复运行中

（2）主机下线

适用场景：服务器下架、不再纳入管控

执行流程：

前置校验：非运行中主机需先停止

优雅停止 Agent 服务

清理主机部署文件

状态更新为已下线

保留数据库记录，支持重新上线

四、批量操作核心优化（生产必备）

异步任务化：批量操作不阻塞页面，后台执行，支持查看进度

部分成功机制：批量操作中，成功主机执行，失败主机标记异常并提示原因

幂等性保障：重复执行同一操作无副作用（如重复启动已运行主机）

操作范围：支持按 IP、分组、状态筛选批量执行

中断与重试：支持终止批量任务，失败节点支持一键重试

五、数据库设计（核心表结构）

数据库使用cloud-server下的resource下的v1.sql

新增主机操作日志表 host\_operation\_log

记录所有增删改查、启停维护操作，用于审计与问题排查

六、补充能力（优化新增，保障可用性）

心跳检测：Agent 定时上报心跳，超时自动标记为异常

操作审计：全量记录操作人、时间、IP、内容、结果

异常告警：部署失败、心跳断开、操作超时自动告警

权限控制：区分管理员 / 操作员，细粒度控制操作权限

结果回调：所有远程操作支持超时机制，避免卡死

分组管理：支持主机按业务 / 环境分组，方便批量管理

七、操作状态流转规则（关键约束）

只有运行中可执行停止 / 重启 / 维护

只有已停止可执行启动

只有已下线/异常可执行删除

维护中主机禁止业务操作，仅允许退出维护 / 下线

所有操作必须校验状态，非法操作直接拦截并提示
